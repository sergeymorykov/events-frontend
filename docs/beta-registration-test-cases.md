# Beta Registration Test Cases

## Интеграционные/e2e кейсы

1. **Submit без чекбокса невозможен**
   - Предусловие: `VITE_BETA_CONSENT_REQUIRED=true`.
   - Шаги:
     - Открыть `/register`.
     - Заполнить `nickname` и `name`.
     - Не отмечать чекбокс согласия.
   - Ожидание:
     - Кнопка `Принять участие` неактивна.
     - Если отправка форсируется, показывается ошибка: `Для регистрации необходимо согласие с условиями бета-тестирования`.

2. **С чекбоксом регистрация успешна**
   - Предусловие: `VITE_BETA_CONSENT_REQUIRED=true`.
   - Шаги:
     - Открыть `/register`.
     - Заполнить `nickname` и `name`.
     - Отметить чекбокс согласия.
     - Нажать `Принять участие`.
   - Ожидание:
     - В `POST /auth/register` уходит payload с `terms_accepted: true`.
     - После успеха отображается сообщение об успешной регистрации и происходит переход на `/`.

3. **Backend 400 корректно отображается в форме**
   - Предусловие: `VITE_BETA_CONSENT_REQUIRED=true`.
   - Шаги:
     - Замокать ответ `POST /auth/register` со статусом `400` и `detail`.
     - Отправить форму.
   - Ожидание:
     - Текст из `detail` отображается в форме ошибки.
     - Не показывается общий текст вроде `Что-то пошло не так`.
     - Состояние чекбокса и введённые поля сохраняются.
